# Сумма отрицательных элементов между экстремумами

Кейс-задача: программа на Python, которая находит **сумму отрицательных элементов**, расположенных **строго между** максимальным и минимальным элементами массива.

---

## Описание действий при выполнении кейс-задачи

### 1. Постановка задачи

- Дан массив (список) чисел.
- Требуется найти **максимальный** и **минимальный** элементы.
- Вычислить сумму **только отрицательных** элементов, которые стоят **между** этими экстремумами (не включая сами максимум и минимум).
- Если максимум и минимум стоят рядом или совпадают, считаем, что между ними нет элементов — сумма равна 0.

### 2. Анализ и выбор подхода

- Решение разбито на этапы: поиск экстремумов → определение границ диапазона → суммирование отрицательных в этом диапазоне.
- Для поиска индексов максимума и минимума использован **один проход** по массиву с `enumerate()` — не нужно проходить массив несколько раз.
- Границы «между» заданы так: левая граница — меньший из индексов max/min, правая — больший. Элементы между ними — с индекса `left_bound + 1` до `right_bound - 1` (строго между).

### 3. Реализация

**Шаг 1. Проверка размера массива**

- Если в массиве меньше двух элементов, между «максимумом» и «минимумом» нет диапазона. Функция возвращает 0 и поясняющее сообщение.

**Шаг 2. Поиск индексов максимума и минимума**

- Инициализация: первый элемент считаем и максимумом, и минимумом, запоминаем его индекс.
- В цикле `for i, val in enumerate(arr)` обновляем `max_val`/`idx_max` и `min_val`/`idx_min` при нахождении большего или меньшего значения.
- В результате получаем индексы `idx_max` и `idx_min`.

**Шаг 3. Определение границ диапазона**

- `left_bound = min(idx_max, idx_min)` — левая граница (ближе к началу массива).
- `right_bound = max(idx_max, idx_min)` — правая граница (ближе к концу).
- Если `right_bound - left_bound <= 1`, между экстремумами нет элементов — возвращаем 0 и информативное сообщение.

**Шаг 4. Суммирование отрицательных элементов**

- В цикле перебираем индексы от `left_bound + 1` до `right_bound - 1` (строго между).
- Если `arr[i] < 0`, добавляем элемент к сумме и при необходимости сохраняем его в список для вывода в описании результата.

**Шаг 5. Формирование результата**

- Функция возвращает два значения: саму сумму и строку с описанием (максимум, минимум, индексы, диапазон, перечень отрицательных элементов в диапазоне).

### 4. Демонстрация работы

- В блоке `if __name__ == "__main__"` генерируется случайный массив заданного размера (например, от 5 до 15 элементов, числа от -50 до 50).
- Выводится исходный массив, затем вызывается `solve_task(A)`.
- На экран выводятся описание и итоговая сумма отрицательных элементов между экстремумами.

### 5. Использование

- **Как модуль:** импортировать `solve_task` и передать список чисел:
  ```python
  from sum_negative_between_extrema import solve_task
  result, info = solve_task([3, -1, 5, -2, 4, -3, 1])
  print(result, info)
  ```
- **Запуск примера:** выполнить файл:
  ```bash
  python sum_negative_between_extrema.py
  ```
